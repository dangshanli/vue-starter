以nfs为pv条件下，单broker kafka的压力测试

# ★配置条件

# kafka配置

  • 部署节点：k8s node节点——10.0.108.12

  • broker数量：1，单节点

  • 垃圾收集器：G1，-XX:+UseG1GC

  • 最大堆内存申请:1G，-Xms1G

  • 持久化volume配置:

spec:
  capacity:
    storage: 20Gi
  accessModes:
    - ReadWriteMany
  storageClassName: kafka-data
  persistentVolumeReclaimPolicy: Retain
  nfs:
    path: /data/k8s/kafka
    server: 10.0.108.11

# zookeeper配置

同样部署在k8s平台的12 node节点，与kafka使用同样的nfs服务。由于其只涉及到集群元信息的存储和broker调度，这里不相干，不在赘述。

# ★压力测试

# 节点IO测试

简单的测试节点的磁盘读写能力：

hdparm -t --direct /dev/sda3;
sync;
time dd if=/dev/zero of=test bs=512k count=10000;

小结：

  • 使用hdparm测试读取，约3760.79 MB/sec。

  • 使用dd测试写入能力,约738 MB/s。

# kafka生产消息测试

kafka-producer-perf-test.sh --topic kcsplog --record-size 512 --num-records 100000 --throughput -1 --producer-props bootstrap.servers=localhost:9092

简单解释：

  • 记录大小：512kb

  • 记录数：10万

  • 不限流

  • pod内部测试，不走端口转发

结果:

75093 records sent, 14338.9 records/sec (7.00 MB/sec), 916.3 ms avg latency, 2914.0 ms max latency.
100000 records sent, 15651.901706 records/sec (7.64 MB/sec), 1638.13 ms avg latency, 3971.00 ms max latency, 656 ms 50th, 3907 ms 95th, 3961 ms 99th, 3969 ms 99.9th.

  • 最大吞吐量为15651条/s，7MB/sec

  • 最低平均时延为916ms，最大时延3971ms

建议：内存可以适当加大到2~4G.
